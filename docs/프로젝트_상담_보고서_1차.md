## 2. 프로젝트 선정 배경 및 동기

### 2.1 기존 게임 개발의 문제점

현재 모바일 센서 게임 개발은 다음과 같은 한계가 있습니다:

| 문제점 | 설명 | 영향 |
|--------|------|------|
| **높은 진입 장벽** | JavaScript, HTML5 Canvas, 센서 API 등 전문 지식 필수 | 비개발자는 아이디어가 있어도 구현 불가 |
| **긴 개발 기간** | 기획부터 배포까지 수주~수개월 소요 | 빠른 프로토타이핑 불가능 |
| **복잡한 센서 통합** | iOS/Android 센서 권한, 데이터 처리 등 복잡 | 개발 난이도 증가 |
| **유지보수 부담** | 버그 수정, 기능 추가 시 수동 코딩 필요 | 지속적인 개발 비용 발생 |
| **플랫폼 파편화** | iOS/Android 앱 각각 개발 필요 | 개발 비용 2배 |

**실제 사례**:
- 간단한 "공 굴리기 게임" 하나 만드는데도 최소 2주 이상 소요
- 센서 권한 처리만 iOS 13+, Android 6+ 각각 다르게 구현
- 게임 아이디어 100개가 있어도 구현은 1-2개만 가능

### 2.2 AI 기술의 발전

최근 Large Language Model(LLM) 기술의 발전으로 코드 자동 생성이 가능해졌습니다:

- **Claude Sonnet 4.5**: 64,000 토큰의 긴 컨텍스트로 완전한 게임 코드 생성 가능
- **RAG 시스템**: 400개 문서를 참조하여 정확한 코드 생성
- **자연어 이해**: "공을 기울여서 굴리는 미로 게임" 같은 간단한 설명만으로 구현 가능

### 2.3 모바일 센서 활용의 필요성

스마트폰의 센서는 강력하지만 활용도가 낮습니다:

- **내장 센서**: 가속도계, 자이로스코프, 방향 센서 등 모든 스마트폰에 탑재
- **실제 활용**: 화면 회전, 만보기 등 제한적 사용
- **잠재력**: 게임 컨트롤러, 피트니스 기기, 교육 도구 등 무궁무진

### 2.4 웹 기반 플랫폼의 접근성

앱 설치 없이 웹 브라우저만으로 실행 가능:

- **접근성**: QR 코드 스캔만으로 즉시 플레이
- **호환성**: iOS, Android 모두 지원
- **배포**: App Store/Play Store 심사 불필요

## 3. 프로젝트 목표 및 핵심 가치

**주요 목표**:
1. 비개발자도 30-60초 내에 센서 게임을 생성할 수 있는 AI 시스템 구축
2. QR 코드만으로 즉시 플레이 가능한 웹 기반 플랫폼 제공
3. AI 기반 자동 유지보수 시스템으로 지속적인 품질 개선

**세부 목표**:

- 센서 데이터 전송 지연 50ms 이하 유지
- 동시 접속자 100명 이상 지원

## 5. 시스템 전체 개요

### 5.1 시스템 아키텍처

```
┌─────────────────────────────────────────────────────────────┐
│                       Client Layer                          │
├─────────────────────────────────────────────────────────────┤
│  💻 PC 게임 화면      📱 모바일 센서      👨‍💻 개발자 센터  │
│  (HTML5 Canvas)     (센서 클라이언트)   (AI 게임 생성기)   │
└─────────────────────────────────────────────────────────────┘
                              ↕️
┌─────────────────────────────────────────────────────────────┐
│                   Communication Layer                       │
├─────────────────────────────────────────────────────────────┤
│  ⚡ WebSocket (Socket.IO)    🌐 HTTP REST API (Express)    │
│  - 50ms 센서 전송            - 게임 생성 API               │
│  - 실시간 양방향 통신        - 유지보수 API                │
└─────────────────────────────────────────────────────────────┘
                              ↕️
┌─────────────────────────────────────────────────────────────┐
│                   Application Layer                         │
├─────────────────────────────────────────────────────────────┤
│  🎮 SessionManager        🤖 InteractiveGameGenerator       │
│  - 세션 생성/관리          - AI 대화형 생성                │
│                                                              │
│  🔧 GameMaintenanceManager  📂 GameScanner                  │
│  - 버그 수정/기능 추가      - 게임 자동 스캔               │
└─────────────────────────────────────────────────────────────┘
                              ↕️
┌─────────────────────────────────────────────────────────────┐
│                     External Services                       │
├─────────────────────────────────────────────────────────────┤
│  🧠 Claude Sonnet 4.5    🔍 OpenAI Embeddings  💾 Supabase │
│  - 코드 생성              - 문서 벡터화         - 벡터 DB  │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 3대 핵심 기능

#### 1️⃣ AI 게임 생성기 (InteractiveGameGenerator)

**5단계 실시간 생성 프로세스**:

```
Step 1 (0-20%): 게임 아이디어 분석
    ↓ AI가 사용자 입력 이해 및 요구사항 추출

Step 2 (20-40%): RAG 문서 검색
    ↓ 400개 문서 중 Top-5 유사 문서 검색

Step 3 (40-80%): Claude AI 코드 생성
    ↓ 64K 토큰 컨텍스트로 완전한 게임 코드 생성

Step 4 (80-90%): 코드 검증
    ↓ 자동 품질 검증 (95점 이상 요구)

Step 5 (90-100%): 파일 저장 및 등록
    ↓ 게임 저장 및 QR 코드 생성
```

**특징**:
- 평균 생성 시간: 30-60초
- 성공률: 95% 이상
- 자동 품질 검증으로 버그 최소화

#### 2️⃣ 실시간 센서 통신 시스템

**센서 데이터 플로우**:

```
📱 모바일 센서
    ↓ DeviceMotion/Orientation 이벤트

📡 50ms 쓰로틀링
    ↓ 네트워크 부하 최적화

⚡ WebSocket 전송
    ↓ Socket.IO 실시간 통신

🎮 SessionManager 라우팅
    ↓ 세션 코드로 PC 찾기

💻 PC 게임 화면 업데이트
    ↓ Canvas 렌더링
```

**특징**:
- 초저지연: 50-100ms
- 자동 재연결
- iOS 13+ / Android 6+ 지원

#### 3️⃣ AI 기반 유지보수 시스템

**자동 버그 수정 프로세스**:

```
사용자: "공이 벽에 붙어서 떨어지지 않아요"
    ↓
AI 분석: 충돌 감지 버그 식별
    ↓
자동 수정: 물리 엔진 코드 수정
    ↓
버전 증가: v1.0 → v1.1
    ↓
자동 백업: 이전 버전 보관
```

**특징**:
- 버그 수정 성공률 90% 이상
- 자동 버전 관리 (v1.0, v1.1, v1.2...)
- 이전 버전으로 롤백 가능

### 5.3 사용자 플레이 시나리오

**전체 플로우 (실제 사용 예시)**:

